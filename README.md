## Описание (что изучили)

В ходе выполнения лабораторной работы №20 были изучены основы работы с **DOM (Document Object Model)** в JavaScript. Мы научились находить элементы на странице, изменять их содержимое и стили, создавать новые элементы динамически, а также обрабатывать события пользователя.

Основной акцент был сделан на сравнении подходов к созданию интерфейсов в **JavaScript (DOM)** и **C# (WinForms/WPF)**. Несмотря на различия в синтаксисе и среде выполнения, концепции управления элементами интерфейса во многом схожи.

### Ключевые темы:
1.  **Поиск элементов:** `getElementById`, `querySelector`.
2.  **Манипуляция контентом:** изменение текста (`textContent`) и HTML (`innerHTML`).
3.  **Управление стилями:** работа со свойством `style`.
4.  **События:** добавление обработчиков через `addEventListener`.
5.  **Динамическое создание элементов:** `createElement`, `appendChild`, `remove`.
6.  **Валидация форм:** проверка данных на стороне клиента перед отправкой.

## Сравнение с C# (WinForms/WPF)

Концепция управления интерфейсом в JavaScript через DOM аналогична работе с элементами управления (Controls) в C# WinForms.

| Концепция | C# (WinForms) | JavaScript (DOM) |
| :--- | :--- | :--- |
| **Найти элемент** | `Button myButton = this.Controls["myButton"];` | `const myButton = document.getElementById("myButton");` |
| **Изменить текст** | `label1.Text = "Новый текст";` | `label.textContent = "Новый текст";` |
| **Добавить обработчик** | `button1.Click += HandleClick;` | `button.addEventListener("click", handleClick);` |
| **Создать элемент** | `Button btn = new Button();` | `const btn = document.createElement("button");` |
| **Добавить в контейнер** | `panel1.Controls.Add(btn);` | `panel.appendChild(btn);` |
| **Скрыть элемент** | `button1.Visible = false;` | `button.style.display = "none";` |

### Обработка событий (Events)

Подход к обработке событий также имеет прямые параллели:

*   **C# (WinForms):** Используется механизм делегатов и событий (`+=`).
    ```csharp
    button.Click += (sender, e) => { MessageBox.Show("Клик!"); };
    ```
*   **JavaScript (DOM):** Используется метод `addEventListener`.
    ```javascript
    button.addEventListener("click", () => { alert("Клик!"); });
    ```

### Валидация формы

*   **C# (WinForms):** Проверка полей обычно происходит в событии `Validating` или перед сохранением данных в коде формы.
*   **JavaScript (DOM):** Проверка происходит в обработчике события `submit` у формы или `click` у кнопки. Позволяет предотвратить отправку формы методом `event.preventDefault()`.

## Главные отличия

Несмотря на схожесть логики, платформы имеют фундаментальные различия:

| Аспект | C# WinForms | JavaScript DOM |
| :--- | :--- | :--- |
| **Компиляция** | Да (компилируемый язык) | Нет (интерпретируется браузером) |
| **Типизация** | Строгая статическая | Динамическая |
| **Ошибки** | Вылавливаются на этапе компиляции | Возникают во время выполнения (в браузере) |
| **UI-дизайнер** | Есть (визуальный Drag & Drop в Visual Studio) | Нет (верстка пишется кодом в HTML/CSS) |
| **Платформа** | Преимущественно Windows | Любой браузер (кроссплатформенность) |

### Преимущества DOM перед WinForms:
*    **Кроссплатформенность:** Работает на Windows, macOS, Linux, Android и iOS.
*    **Отсутствие установки:** Пользователю не нужно ничего устанавливать, достаточно браузера.

*    **Современный UI:** Гибкие возможности стилизации через CSS и анимации.

### Преимущества WinForms перед DOM:
*   **Визуальный дизайнер:** Быстрое прототипирование интерфейса мышкой.
*    **Безопасность типов:** Многие ошибки обнаруживаются до запуска программы.
*    **Интеграция с ОС:** Глубокий доступ к функциям Windows API.


> **Вывод:** DOM и WinForms решают схожие задачи по созданию пользовательских интерфейсов, но используют разные подходы, обусловленные средой выполнения (браузер vs операционная система).
